AI 분석 파이프라인 — 상세 설계

6개 서브모듈의 입력/처리/출력 흐름과 TwelveLabs API 호출 명세

3A. SOP 단계 검출 (Step Detection)

Marengo Search × SOP Steps → 수행 타임라인
INPUT
SOP 단계 목록 (pov-standards.ts)
"1.1 냉각수 계통 밸브 VG-003 열림 확인"

자연어 쿼리 변환 규칙
→ "operator checking valve VG-003 open"

videoId (인덱싱 완료된 영상)
→
PROCESS
1) 배치 검색
SOP N단계 × Marengo Search API
(병렬 호출, rate limit 고려)

2) 신뢰도 필터링
confidence ≥ 0.6 → 수행
0.3 ≤ conf < 0.6 → 부분(partial)
conf < 0.3 → 미수행

3) 타임스탬프 정렬
검출된 단계를 시간순 정렬
→
OUTPUT
DetectedStep[]
{ stepId: "1.1", status: "pass"|"fail"|"partial", confidence: 0.87, timestamp: 45.2, // 초 endTime: 52.8, searchScore: 0.87, thumbnailUrl?: string }
TwelveLabs API 호출 예시
POST /search { "index_id": "khnp-pov-training", "query_text": "operator opens valve VG-003 by turning handle", "search_options": ["visual", "audio"], "threshold": { "min": 0.3 }, "sort_option": "score", "page_limit": 5, "filter": { "id": ["video_abc123"] } }

3B. 손-물체 상호작용 분석 (Hand-Object Analysis)

Pegasus Analyze → 도구 식별 + 조작 대상 + 상태 변화
INPUT
videoId + DetectedStep[]
(3A에서 검출된 단계 구간)

분석 프롬프트 템플릿
장비 목록 + 예상 상태 포함

구간별 분석
각 단계의 timestamp~endTime
→
PROCESS
1) 구간별 Pegasus 분석
프롬프트 엔지니어링:
"This is a first-person POV video of a nuclear plant operator. Describe: 1) What tool/object is the operator holding? 2) What equipment are they interacting with? 3) What is the before/after state?"

2) 구조화 파싱
LLM 응답 → JSON 변환

3) SOP 기대값 교차검증
분석 결과 vs 기대 상태 비교
→
OUTPUT
HandObjectEvent[]
{ stepId: "1.1", timestamp: 45.2, heldObject: "렌치", targetEquipment: "VG-003", actionType: "turn_valve", stateBefore: "closed", stateAfter: "open", matchesSOP: true, confidence: 0.82 }
도구 (Held Objects)
렌치, 드라이버, 체크리스트,
키, 무전기, 보호장갑,
계측기, 펜, 클립보드
조작 대상 (Targets)
밸브 핸들, 스위치, 버튼,
계기판, 펌프 레버,
차단기, 제어반
상태 변화 (States)
열림↔닫힘, 기동↔정지,
ON↔OFF, 정상↔비정상,
증가↔감소, 도달↔미도달

3C. 시퀀스 매칭 — DTW 알고리즘

SOP 순서 vs 실제 순서 → 이탈 타임라인
SOP 정의 순서 (Expected)
1.1 → 1.2 → 1.3 → 2.1 → 2.2 → 2.3
실제 수행 순서 (Detected)
1.1 → 1.3 → 1.2 → 2.1 → 2.2 → 2.3
순서 오류
SWAP
1.2 ↔ 1.3
단계 누락
SKIP
2.2 미수행
추가 행동
INSERT
SOP에 없는 행동
시간 이탈
DELAY
기대시간 초과
DTW 출력 구조
interface SequenceAlignment { sopSequence: string[]; // ["1.1","1.2","1.3","2.1","2.2","2.3"] detectedSequence: string[]; // ["1.1","1.3","1.2","2.1","2.3"] alignmentPath: AlignmentPair[]; // DTW 최적 정렬 경로 deviations: SopDeviation[]; // 이탈 목록 complianceScore: number; // 0-100 (순서 준수율) criticalDeviations: number; // 핵심단계 이탈 수 }

3D~3F. HPO 검증 + 임베딩 비교 + 종합 스코어링

3D. HPO 도구 검증
11개 HPO 도구 각각에 대해:

Marengo Search로 적용 장면 검색
"operator performing STAR self-check"
"peer checking procedure step"

출력:
도구별 적용 횟수 + 타임스탬프
기본4종 필수 / 조건부7종 선택
기본 4종 미적용 = 감점
3E. 임베딩 비교
골드스탠다드 영상이 있을 때:

양쪽 영상을 10초 세그먼트로 분할
Marengo Embed → 512d 벡터
세그먼트별 코사인 유사도 계산

출력:
유사도 히트맵 (시간 × 시간)
저유사도 구간 = 격차 구간
전체 평균 유사도 점수
3F. 종합 스코어링
가중 합산 공식:

절차준수 × 0.40
+ HPO도구 × 0.30
+ 기본수칙 × 0.20
+ 유사도 × 0.10
─────────────
= 종합점수 (0~100)

등급 매핑:
S(95+) A(85+) B(70+)
C(55+) D(40+) F(<40)

핵심 기술 과제: 쿼리 엔지니어링

한국어 SOP → 영어 자연어 쿼리 변환 전략

TwelveLabs는 영어 쿼리에서 최고 성능을 발휘합니다. SOP 단계를 효과적인 영어 쿼리로 변환하는 것이 시스템 정확도의 핵심입니다.

BAD Query
"냉각수 계통 밸브 VG-003 열림 확인"
→ 한국어 직역, 너무 추상적
GOOD Query
"operator's hand turning a valve handle to the open position, checking the valve indicator"
→ 1인칭 시점 행동 묘사, 구체적
쿼리 변환 규칙 (pov-standards.ts에 추가)
interface StepQueryTemplate { stepId: string; // SOP 원문 (한국어) sopText: string; // 1차 검색: 행동 기반 (높은 재현율) actionQuery: string; // "hand turning valve handle" // 2차 검색: 물체 기반 (높은 정밀도) objectQuery: string; // "valve labeled VG-003" // 3차 검색: 상태 기반 (검증용) stateQuery: string; // "valve indicator showing open" // 복합 검색: 위 쿼리 조합 composedQuery?: string; // 필요시 이미지+텍스트 }
