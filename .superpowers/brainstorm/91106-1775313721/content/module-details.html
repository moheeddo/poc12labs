모듈 상세 설계 — 10개 기능 매핑

각 모듈의 입출력, 의존성, API 설계

1
evidence/ — 증거 기반 설명 가능성
Gap #1: Explainability
입력
  • 루브릭 기준 텍스트
  • 영상 videoId
  • AI 점수 + 역량 키
처리
  • 루브릭 기준별 TwelveLabs Search
  • 매칭 구간 신뢰도 필터링
  • 증거 클립 타임스탬프 추출
출력
  • EvidenceMap: 점수↔영상구간
  • 클릭 가능한 증거 링크
  • 신뢰도 표시 (%, 색상)
API: /api/evidence/map — POST { videoId, competencyKey, rubricItems[] } → EvidenceMap[]
2
derailer/ — 탈선 요인 탐지
Gap #2: Hogan HDS 기반
입력
  • 영상 transcript
  • 멀티모달 5채널 신호
  • 시나리오 맥락 (비상/일상)
처리
  • 11개 탈선 패턴 프롬프트
  • 음성 긴장/표정 변화 감지
  • 맥락별 가중치 조정
출력
  • DerailerProfile: 11개 위험도
  • 위험 구간 타임스탬프
  • 개발 권고사항
11개 탈선 패턴 (Hogan HDS 원전 맥락화):
과도한 지시형 회피형 변덕형 과잉신중형 독선형 과시형 수동공격형 무관심형 과잉순응형 완벽주의형 고립형
API: /api/derailer/analyze — POST { videoId, transcript, multimodalSignals, scenarioType } → DerailerProfile
3
bei/ — 행동사건면접 자동화
Gap #3: STAR 구조 파싱
입력
  • 영상 transcript (타임스탬프)
  • 역량 프레임워크
  • 직급별 기대 행동
처리
  • 발언 → STAR 구조 파싱
  • 행동사건 단위 분리
  • 역량 코딩 (자동 분류)
출력
  • BEIEvent[]: STAR 구조화
  • 역량별 행동사건 매핑
  • 차별화 역량 식별
API: /api/bei/extract — POST { videoId, transcript, competencyFramework } → BEIEvent[]
API: /api/bei/code — POST { events[], competencies[] } → CompetencyCoding[]
4
growth/ — 역량 성장 추이 추적
Gap #4: Longitudinal Tracking
입력
  • 과거 세션 평가 데이터
  • 현재 세션 평가 데이터
  • Video Embeddings (과거/현재)
처리
  • 점수 시계열 분석
  • Embeddings 코사인 유사도
  • 성장률 계산 + 추세선
출력
  • GrowthTimeline: 역량별 곡선
  • 유사도 변화 그래프
  • 개선 영역/정체 영역 식별
API: /api/growth/timeline — POST { employeeId, sessions[] } → GrowthTimeline
API: /api/growth/similarity — POST { videoId1, videoId2 } → SimilarityReport
5
validation/ — 심리측정 타당화 + 노름
Gap #5 + #6: Psychometric Validation + Norms
입력
  • 전체 평가 데이터셋
  • 인간 평가자 점수
  • 인사DB 메타데이터
처리
  • Cronbach's α 계산
  • ICC(AI vs 인간) 산출
  • 직급/부서별 노름 생성
출력
  • ValidationReport: 신뢰도 지표
  • NormTable: 백분위 테이블
  • 적정 표본 크기 안내
API: /api/validation/reliability — POST { dataset } → ReliabilityReport
API: /api/validation/norms — POST { dataset, groupBy } → NormTable
6
fairness/ — 편향/공정성 모니터링
Gap #7: Bias Monitoring
입력
  • 평가 점수 + 인구통계
  • 인사DB (성별/연령/경력)
  • 역량별 문항 응답 데이터
처리
  • 집단 간 점수 분포 비교
  • DIF 분석 (Mantel-Haenszel)
  • 4/5 규칙 자동 적용
출력
  • FairnessReport: 편향 지표
  • DIF 의심 항목 플래그
  • 불리한 영향 경고 알림
API: /api/fairness/analyze — POST { scores[], demographics[] } → FairnessReport
API: /api/fairness/dif — POST { itemResponses[], groupVar } → DIFResult[]
7
compliance/ — ISO 10667 + 삼각측정 + 실시간 코칭
Gap #8 + #9 + #10: Integration & Compliance
삼각측정 (Gap #8)
  • AI 영상분석 점수
  • 인간 관찰평가 점수
  • 가중 융합 → 최종 점수
ISO 10667 (Gap #9)
  • 참여자 동의 관리 UI
  • 감사 로그 (모든 평가 행위)
  • 데이터 보존/삭제 정책
실시간 코칭 (Gap #10)
  • 평가 직후 즉시 피드백
  • 이전 피드백 대비 개선 매핑
  • 다음 개발 과제 자동 제안
API: /api/compliance/triangulate — POST { aiScore, humanScore, weights } → FinalScore
API: /api/compliance/consent — POST { participantId, consentType, agreed } → ConsentRecord
API: /api/compliance/audit-log — GET { sessionId } → AuditEntry[]
+
hr-connector/ — 인사 DB 연계
Shared Infrastructure (신규)
제공 데이터
  • 사원번호, 성명, 부서, 직급
  • 성별, 연령, 입사년도, 경력
  • 조직도 (상사/동료/부하 관계)
  • 과거 평가 이력 (연계 가능 시)
연동 방식 (설정 가능)
  • 모드 A: CSV/Excel 일괄 업로드
  • 모드 B: REST API 연동 (실시간)
  • 모드 C: LDAP/AD 연동
  • 개인정보 마스킹 + 암호화 저장
사용처: validation(노름 구축), fairness(편향 분석), compliance(동의 관리), growth(개인 추적)
