# 교수자 강의평가 시스템 설계 (Lecture Evaluation)

> 교육 컨텐츠 환류 체계의 핵심 — 강의 전달력 + 강의안 대비 내용 충실도를 AI로 정량 평가

## 1. 서비스 개요

| 항목 | 값 |
|------|------|
| 서비스명 | 교수자 강의평가 (Lecture Evaluation) |
| 탭 컬러 | Coral (`--color-coral-*`) |
| TwelveLabs 인덱스 | `khnp-lecture-evaluation` |
| 단축키 | `3` |
| ServiceTab 키 | `"lecture"` |

### 핵심 컨셉: 듀얼 엔진 아키텍처

내용 충실도 엔진과 전달력 엔진을 분리하고, 최종 통합 스코어링.

- **내용 충실도 엔진**: PPT 슬라이드 노트(강의스크립트) vs 실제 강의 전사 비교
- **전달력 엔진**: 멀티모달 5채널(기존 재활용) + 교수법 특화 3지표(신규)

## 2. 입력물 및 분석 모드

### 입력물

| 입력 | 형태 | 필수 여부 |
|------|------|-----------|
| 강의 영상 | MP4/AVI/MOV | 필수 |
| 강의안 PPT | .pptx (슬라이드 노트 포함) | 선택 |
| 강의 정보 | 교수자명, 과목명, 일시 | 필수 |

### 분석 모드

| 모드 | 입력물 | 분석 범위 | 총점 |
|------|--------|-----------|------|
| 영상 Only | 강의 영상만 | 전달력 엔진만 | 전달력 50점 → 100점 환산 |
| 영상 + PPT | 강의 영상 + 강의안 | 전달력 + 내용 충실도 | 100점 (전달력 50 + 내용 50) |

PPT 미첨부 시 내용 충실도 섹션은 "강의안 첨부 시 분석 가능" 안내.
PPT를 나중에 추가 첨부하면 내용 충실도만 추가 분석 (전달력 재분석 불필요).

## 3. 데이터 흐름

```
[교수자/관리자]
    │
    ├── 강의 영상 업로드 → TwelveLabs 인덱싱 → videoId
    │
    └── 강의안 PPT 업로드 → 서버 파싱 ─┬→ 슬라이드 목록 (제목+본문)
                                        └→ 슬라이드 노트 (강의스크립트)
    │
    ▼ [분석 시작]
    │
    ├── 전사 추출 (TwelveLabs transcription API)
    │
    ├── ┌─ 내용 충실도 엔진 ──────────────────────┐
    │   │ Layer 1: 슬라이드별 커버리지 (0~100%)    │
    │   │ Layer 2: 핵심 개념 의미론적 매칭          │
    │   │ Layer 3: 전달 품질 평가                   │
    │   └──────────────────────────────────────────┘
    │
    ├── ┌─ 전달력 엔진 ───────────────────────────┐
    │   │ 기존 5채널 (시선/음성/유창성/자세/표정)    │
    │   │ + 교수법 확장 지표 3개                     │
    │   └──────────────────────────────────────────┘
    │
    ▼
    통합 스코어링 → Solar 리포트 생성
    │
    ├── 교수자용: 개선 코칭 리포트
    └── 관리자용: 종합 현황 대시보드
```

## 4. 내용 충실도 엔진 (3계층)

PPT가 첨부된 경우에만 작동.

### 4.1 PPT 파싱

서버 API `/api/lecture/parse-ppt`에서 .pptx ZIP 내 XML을 직접 파싱:

```typescript
interface ParsedSlide {
  index: number;           // 슬라이드 번호 (1-based)
  title: string;           // 슬라이드 제목
  bodyText: string;        // 본문 텍스트
  notes: string;           // 슬라이드 노트 = 강의스크립트
  keyConcepts?: string[];  // LLM이 추출한 핵심 개념
}

interface ParsedPpt {
  fileName: string;
  slideCount: number;
  slides: ParsedSlide[];
  totalNotesLength: number; // 전체 노트 길이 (충실도 기준)
}
```

### 4.2 Layer 1: 슬라이드별 커버리지

전사 텍스트를 슬라이드 순서대로 매칭하여 각 슬라이드의 내용이 강의에서 언급되었는지 정량화.

- 방법: TwelveLabs `generate` API에 슬라이드 노트 + 전사를 넣고, 슬라이드별 커버리지 % + 근거 구간(타임스탬프)을 JSON으로 추출
- 출력: `{ slideIndex, coveragePercent, matchedSegments: [{ start, end, text }] }`
- 배점: 15점 (전체 커버리지 평균)

### 4.3 Layer 2: 핵심 개념 의미론적 매칭

각 슬라이드 노트에서 핵심 개념/용어를 추출하고, 전사에서 해당 개념이 의미적으로 언급되었는지 판단.

- 방법:
  1. 슬라이드 노트 → LLM으로 핵심 개념 추출 (개념명 + 정의)
  2. TwelveLabs search로 각 개념 검색
  3. 매칭 결과 집계 (found / partial / missing)
- 출력: `{ concept, status: "found"|"partial"|"missing", timestamp?, evidence? }`
- 배점: 20점 (매칭률 기반)

### 4.4 Layer 3: 전달 품질 평가

매칭된 개념에 대해 얼마나 잘 설명했는지 평가.

| 전달 품질 지표 | 설명 | 점수 |
|---------------|------|------|
| 예시 활용 | 추상 개념에 구체적 사례를 들었는가 | 0-3 |
| 반복/요약 | 핵심 내용을 반복하거나 요약 정리했는가 | 0-3 |
| 학습자 상호작용 | 질문 유도, 확인 질문 등 | 0-3 |
| 논리적 연결 | 이전 슬라이드 내용과 자연스러운 브릿지 | 0-3 |

- 방법: 개념별 전사 구간을 TwelveLabs `generate`에 넣고 4개 지표를 0-3점으로 평가
- 배점: 15점 (4지표 합산 환산)

## 5. 전달력 엔진

### 5.1 멀티모달 5채널 (기존 재활용)

| 채널 | 지표 3개 | 교수자 맥락 해석 |
|------|---------|-----------------|
| 시선 (Gaze) | 청중 응시율, 시선 이탈 빈도, 하방 응시율 | 학습자를 골고루 보는가 |
| 음성 (Voice) | F0 변화폭, 음량 변화, 강조 빈도 | 단조롭지 않은 강의 톤인가 |
| 유창성 (Fluency) | 조음 속도, filled pause, 장무음 | 매끄럽게 진행하는가 |
| 자세 (Posture) | 개방 자세, 목적형 제스처, 잔동작 | 자신감 있는 강의 자세인가 |
| 표정 (Face) | 몰입 표정 비율, 긴장도, 급격한 움직임 | 편안하고 친근한 표정인가 |

기존 `multimodal-scoring.ts` 엔진을 공통 모듈로 사용. 리더십/강의평가 양쪽에서 호출.
배점: 35점 (0-9 스케일을 35점으로 변환)

### 5.2 교수법 특화 확장 지표 3개 (신규)

| 확장 지표 | 추출 방법 | 채점 기준 |
|----------|----------|----------|
| 학습자 호명/질문 유도 | 전사에서 패턴 탐지 ("~씨", "질문 있으신 분", "어떻게 생각하세요") | 빈도 + 분포 균등성 |
| 판서/슬라이드 포인팅 | TwelveLabs generate로 포인팅 장면 검색 | 핵심 슬라이드에서 포인팅 여부 |
| 요약/전환 시그널 | 전사에서 메타 담화 탐지 ("정리하면", "다음으로", "지금까지") | 슬라이드 전환 시 브릿지 여부 |

별도 API 라우트 `/api/lecture/pedagogy-extract`로 구현.
배점: 15점 (3지표 각 5점)

## 6. 통합 스코어링

```
강의평가 종합 점수 (100점)
├── 전달력 (50점)
│   ├── 멀티모달 5채널 (35점) ← 기존 0-9 → 35점 변환
│   └── 교수법 확장 3지표 (15점) ← 각 5점
│
└── 내용 충실도 (50점) ── PPT 미첨부 시 전달력만 100점 환산
    ├── 슬라이드 커버리지 (15점)
    ├── 핵심 개념 매칭 (20점)
    └── 전달 품질 (15점)
```

### 등급 기준

| 등급 | 점수 범위 | 설명 |
|------|----------|------|
| 탁월 | 90-100 | 모범 강의 |
| 우수 | 75-89 | 효과적 강의 |
| 보통 | 60-74 | 개선 필요 |
| 미흡 | 0-59 | 집중 코칭 필요 |

## 7. UI 구조

### Phase 기반 상태 머신

```
[Phase 1] 업로드
  ├── 강의 영상 업로드 (필수) — 기존 VideoUploader 재사용
  ├── 강의안 PPT 업로드 (선택) — PptUploader 컴포넌트
  └── 강의 정보 입력: 교수자명, 과목명, 일시

[Phase 2] 분석 중
  ├── 진행률 UI (기존 POV 패턴 재사용)
  └── 7-스테이지 표시:
      ① 영상 인덱싱 → ② 전사 추출 → ③ PPT 파싱
      → ④ 내용 충실도 분석 → ⑤ 전달력 분석
      → ⑥ 통합 스코어링 → ⑦ 리포트 생성

[Phase 3] 결과
  ├── 탭 1: 종합 대시보드 (점수 + 등급 + 요약)
  ├── 탭 2: 내용 충실도 상세 (PPT 있을 때만)
  │   ├── 슬라이드 맵 (커버리지 히트맵)
  │   ├── 핵심 개념 매칭 테이블
  │   └── 전달 품질 카드
  ├── 탭 3: 전달력 상세
  │   ├── 8축 레이더 차트 (5채널 + 교수법 3지표)
  │   ├── 채널별 코칭 코멘트
  │   └── 타임라인 하이라이트
  └── 탭 4: 이력/추이
      ├── 과거 평가 목록
      └── 시간순 점수 추이 라인차트
```

### 교수자용 코칭 리포트

- 종합 점수 + 등급 배지
- 내용 충실도 슬라이드 맵 (커버리지 바 차트)
- 미전달 슬라이드 경고 + 구체적 개선 안내
- 전달력 8축 레이더 차트 + 채널별 코칭
- 타임라인 하이라이트 (좋은 순간 / 개선 순간)

### 관리자용 대시보드 (MVP는 이력 테이블)

- 교수자별 점수 비교 테이블
- 전체 평균 대비 개인 위치
- 시간순 추이 (동일 교수자 복수 평가 시)
- 약점 패턴 요약

## 9. 분석 파이프라인 스테이지

```
Stage 1: 전사 추출            ← 기존 /api/twelvelabs/transcription
Stage 2: PPT 파싱             ← /api/lecture/parse-ppt (PPT 있을 때만)
Stage 3: 내용 충실도 분석      ← 3계층 순차 (PPT 있을 때만)
Stage 4: 멀티모달 5채널        ← 기존 /api/twelvelabs/multimodal-extract
Stage 5: 교수법 3지표          ← /api/lecture/pedagogy-extract
Stage 6: 통합 스코어링         ← lecture-scoring.ts
Stage 7: Solar 리포트          ← 기존 /api/solar/report (프롬프트만 교수자용으로)
```

Job 패턴: POV와 동일한 비동기 시작 → 폴링 방식.

## 10. 향후 확장 (Phase 2+)

- 슬라이드 전환 시점 자동 탐지 (OCR/프레임 비교) → 슬라이드 단위 정밀 분석
- 학습자 반응 분석 (교실 전체 영상에서 학습자 집중도 추정)
- 관리자 대시보드 고도화 (교수자 간 비교, 조직 차원 약점 분석)
- 교수법 루브릭 커스터마이징 (과목/분야별 가중치 조정)
